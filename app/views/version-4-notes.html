Version 4 - tested in Belfast on 21 February 2020

What we wanted to test

In this version we tested the case list design in more depth.

We tested with 6 PIP healthcare professionals at Capita.

Changes since version 3

Changes since version 3:

  • totals are less prominent on the dashboard
  • users are taken straight to the ‘Details’ tab rather than the ‘Timeline’ tab - this is what happens in the service now but it also helps us to see if it clarifies at what point they'd click on 'make recommendation'
  • added a list of reasons for saving (which will appear as labels on the case list) and a couple of different options for setting how long it will be saved
  • made the confirmation messages more obvious - rather than a banner at the top of the case list they are now taken to a page with a large green banner and a button to take them back to the case list
  • changed the labels on the dashboard into more descriptive tasks to see if that helped with comprehension
  • removed any reasons for returning to DWP that didn’t seem right last time
  • changed ‘Return to DWP’ to ‘Suggest return to DWP’ to see if that made healthcare professionals more comfortable with taking action
  • made some of the plain English text clearer - for example, ‘Changed assessment route’ is now ‘New evidence received’
  • changed the hint text on the case list so it doesn't contradict the saved labels
